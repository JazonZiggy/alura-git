Especificacoes de repositorio

.gitignore

        Arquivo onde se especifica quais arquivos na pasta devem ser ignorados pelo git

Comandos git

git init

Cria um repositorio a partir de uma pasta

Parametros
    --bare  - Com este comando nós criamos um repositório que não terá a working tree, ou seja, 
    não conterá uma cópia dos nossos arquivos. Como o repositório servirá apenas como servidor,
    para que outros membros da equipe sincronizem seus trabalhos, poupamos espaço de armazenamento desta forma.

git status

Verifica o status dos arquivos na pasta

git add "nome do arquivo"

   
        Adiciona o arquivo informado como parametro para ser acompanhado pelo git
    Tambem pode ser passado o parametro . para adicionar a pasta toda

git rm "nome do arquivo"

        Remove um arquivo do acompanhamento do git

git commit -m "Mensagem do commit"

        Faz um commit dos arquivos que foram modificados na pasta que o git add esta acompanhando.
    A mensagem passada após o parametro -m é para ser bem descritiva do motivo do commit

git push "nomedoRepositorioRemoto" "branch a 'ser empurrada'"

        Faz o upload do repositorio local para o repositorio remoto
    Os parametros que foram passados são os nomes dos repositorios remotos e a branch que sera feito o 
    upload.

git pull "nomedoRepositorioRemoto" "branch a ser feito o download"

        Faz o download dos arquivos de um determinado repositorio

git clone "pasta de um projeto ou url"

        Baixa um projeto para a pasta local passando um parametro que pode ser a pasta ou url

git remote

        Lista todos os repositorios remotos que o repositorio local conhece

git branch

        Lista todos os branches
        Pode ser adicionado um parametro com um "nome" para criação de um novo branch

git checkout

Parametros
        git remote add nome-repositorio caminho/para/o/repositorio
        Desta forma teremos um link do nosso repositório local com o repositório remoto,
        que chamamos de nome-repositorio, que está armazenado em caminho/para/o/repositorio.
    
        git remote rename nomeAtual nomeNovo 
        Renomeia o nome do repositorio Remoto.

git log

        Histórico de commits, pode ser adicionado parametros como o --oneline
    para demonstrar os commits em somente uma linha.
        Outro parametro que pode ser utilizado é o -p, que demonstra com detalhes
    o que foi modificado
        --grep tambem pode ser utilizado informando somente o valor a ser procurado
        O comando --pretty pode ser utilizado com o comando format % para adicionar informaçoes
    No formato que deseja ser visualizado.

git config --local user.name "Seu Usuario"

        Adiciona o usuario as configurações locais do git

git config -- local user.email "E-mail"

        Adiciona o e-mail as configurações locais do git

Definiçoes

Head

        Estado Atual do nosso código, ou seja, onde o Git os colocou

Working Tree

        Local onde os arquivos realmente estão sendo armazenados e editados

index

        Local onde o Git armazena o que será comitado, ou seja, o local entre a working tree e o 
    repositório git em si.
